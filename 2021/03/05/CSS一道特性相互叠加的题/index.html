问：
position跟display、margin collapse、overflow、float这些特性相互叠加后会怎样？ 这个问题可以拆成两个方面：

  • ‘display’、’position’ 和 ‘float’ 的相互关系；
  • position跟display、overflow、float下的margin collapse

总的来说，可以把它看作是一个类似优先级的机制

  • display: none; 的优先级最高；
  • 然后是 “position: absolute/fixed”，有它存在的时候，浮动不起作用，’display’的值也需要调整；
  • 其次，元素的’float’特性的值不是”none”的时候，调整’display’的值
    float为none，但它是根元素的时候，调整’display’的值；
  • 最后，非根元素，并且非浮动元素，并且非绝对定位的元素，’display’特性值同设置值。

二、position跟display、overflow、float下的margin collapse

margin collapse 即外边距合并，指的是毗邻的两个或多个外边距 (margin) 会合并成一个外边距。

其中所说的 margin 毗邻，可以归结为以下两点：

  • 这些 margin 没有被非空内容、padding、border 或 clear 分隔开。
  • 这些 margin 都处于普通流中。
  1. 两个或多个毗邻的普通流中的块元素垂直方向上的 margin 会折叠。

需要注意的一点是，在没有被分隔开的情况下，一个元素的 margin-top 会和它普通流中的第一个子元素(非浮动元素等)的 margin-top 相邻； 只有在一个元素的 height 是 “auto” 的情况下，它的 margin-bottom 才会和它普通流中的最后一个子元素(非浮动元素等)的 margin-bottom 相邻。例子如下：

一、margin 合并问题

在垂直方向上, margin值合并问题
如果两个同级的块级元素(两个盒子是上下关系),上边的一个盒子设置 margin-bottom,下面的一个盒子设置 margin-top
两个盒子之间的距离 并不是他们的margin之和,而是取最大的那个值。 我们尽量在垂直方向上写一致就可以了,只设置margin-top或者设置margin-bottom就行了,不要两个都写

答案：15px
相邻元素的 margin-top 和 margin-bottom 会发生重叠（必须记住）
空白内容的也会重叠

在实际应用时,在margin合并这个问题上,我们一般不用bfc解决问题（因为要给元素加一层父元素，改变了HTML结构）,而是通过只设置上面的元素的margin-bottom来解决距离的问题

二、margin 塌陷问题

描述：大盒子里面嵌套小盒子,如果这个父盒子里面没有文字，也没有边框（border），也没有padding-top （这三个条件中没有一个满足的）,给小盒子添加 margin-top 值,不但没有实现和大盒子之间的间距,反而传递到大盒子上,导致整体下移，这个现象就叫做margin值穿透。

margin塌陷解决方法

  1. 给父级设置边框或内边距(不建议使用)
    这种方法会影响最后实际的高度，需要再高度height的基础上减掉,才不会影响最后实际的高度

  2. 给子元素的 margin-top 值改为给父元素加 padding-top 这样就避免了使用 margin-top 值导致的传递问题(转移了我们的目标对象)

  3. 触发bfc(块级格式上下文),改变父级的渲染规则(最重要)

    改变父级的渲染规则有以下四种方法,给父级盒子添加
    (1)position:absolute/fixed
    (2)display:inline-block;
    (3)float:left/right;
    (4)overflow:hidden; (最常用)
    这四种方法都能触发 bfc ,但是使用的时候都会带来不同的麻烦,具体使用中还需根据具体情况选择没有影响的来解决margin塌陷

三、给行内元素设置margin

行内元素设置 margin 会有位置的改变吗?
答：左右 margin 会变，上下 margin 不会变。
对于一个行内元素，设置它的 margin 会有什么表现?
答：设置 margin-left 和 margin-right 有影响，而 margin-top 和 margin-bottom 无影响。

🚩 行内元素中，padding-left / padding-right / margin-left / margin-right 有影响结果；
🚩 行内元素中，padding-top / padding-bottom / margin-top / margin-bottom 不影响结果；
🚩 padding-top / padding-bottom 虽然不影响结果，但实际上生效了。

当没有 background-color: red 时，结果是 span 向右移动了 50px+10px=60px 的距离，即：
padding-left / padding-right / margin-left / margin-right 影响了结果(水平方向上的)
padding-top / padding-bottom / margin-top / margin-bottom 不影响结果(垂直方向上的)

加上 background-color: red 后，我们发现其实 padding-top / padding-bottom 生效了，不过还是没影响到结果；而 margin-top / margin-bottom 没有生效。

出现上述情况的原因

span 是 **不可替换元素 (non-replaced element)**，而 行内不可替换元素的外边距不会改变该元素的行高。
关键词：行内 不可替换元素 行高

因为 <span> 既是行内元素，又是不可替换元素。而行内元素中，不可替换元素的外边距不会改变该元素的行高，所以在 <span> 中当我们试图使用 margin 或 padding 改变元素的边距时，行高是不变的，只是宽度变大了。

假如把 <span> 换成可替换元素，改变边距就会影响行高了？
是的。

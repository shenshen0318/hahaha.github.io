Vuex 是什么？Vuex 的核心概念？为什么要用 Vuex？

Vuex 是什么？

vuex 是一个专为 vue.js 应用程序开发的状态管理模式, 采用集中式存储管理所有组件的状态。vuex 本质就是一个仓库，仓库里放了很多对象。
就是把多个组件要共享的变量全部存储在一个对象里面，把这个对象放在顶层的 Vue 实例中，让其他组件可以使用。
Vuex 是响应式的。

什么状态需要在多个组件中共享？（Vuex 应该存储哪些）
目前主要有两种数据会使用 vuex 进行管理：
1、组件之间全局共享的数据

  • 比如用户的登陆状态（token），用户名称，头像，地理位置信息等等；

2、通过后端异步请求的数据 比如做加入购物车、登录状态等

  • 比如商品的收藏，购物车里的商品等等；

怎么使用？
用来读取的状态集中放在 store 中；改变状态的方式是提交 mutations，这是个同步的事物；异步逻辑应该封装在 action 中。
在 main.js 引入 store，注入。新建了一个 目录store，… export

Vuex的核心概念

vuex 五大核心属性：state，getter，mutation，action，module

  • 通过 this.$store.state.属性 的方式来访问状态
  • 通过 this.$store.commit(‘mutation中的方法’) 来修改状态

Vuex 的数据传递流程：
当组件进行数据修改的时候我们需要调用 dispatch(异步操作) 来触发 actions 里面的方法。actions 里面的每个方法中都会有一个 commit(同步操作) 方法，当方法执行的时候会通过 commit 来触发 mutations 里面的方法进行数据的修改。mutations 里面的每个函数都会有一个 state 参数，这样就可以在 mutations 里面进行 state 的数据修改，当数据修改完毕后，会传导给页面。页面的数据也会发生改变。

dispatch：异步操作，写法：this.$store.dispatch(‘actions方法名’,值)

commit：同步操作，写法：this.$store.commit(‘mutations方法名’,值)

  • state：单一状态树，又称单一数据源。用来存储数据、存储状态，对应于与一般 Vue 对象里面的 data。state 里面存放的数据是 响应式 的，Vue 组件从 store 中读取数据，若是 store 中的数据发生改变，依赖这个数据的组件也会发生更新。Vuex 使用单一状态树来管理应用层级的全部状态,（即每个应用将仅仅包含一个 store 实例），但单一状态树和模块化并不冲突，单一状态树能够让我们用最直接的方式找到某个状态的片段，易于维护与调试。

  • getter：类似 vue 的计算属性（computed）。当需要从 store 中获取一些 state 变形之后的状态时用 getters。getters 对比 computed 的好处是 所有组件都能使用，代码复用。getters 接受的参数只有 state 和 getters，若想要传递其他参数，只能让 getters 返回一个函数，在函数里面传递参数。

  • mutation：Vuex 中 store 状态（state）更新的唯一方式是提交 mutations。Vuex 要求 mutations 中的方法必须是同步方法，主要原因是如果是一步操作，devtools 将不能很好的追踪这个操作什么时候会完成。
    mutation 主要包括两个部分：

    • 事件类型（type）
    • 回调函数（第一个参数就是 state）

  • action：类似与 mutation，用于代替 mutation 进行异步的操作，通过提交 mutation 间接更变状态。

  • module：Vuex 使用单一状态树，store 对象就有可能变得相当臃肿，为了解决这个问题，Vuex 允许将 store 分割成模块（module），而这个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块。
    使用：this.$store.state.module名.属性

为什么要用 Vuex

由于传参的方法对于多层嵌套的组件将会非常繁琐，并且对于兄弟组件间的状态传递无能为力。我们经常会采用父子组件直接引用或者通过事件来变更和同步状态的多份拷贝。以上的这些模式非常脆弱，通常会导致代码无法维护。所以我们需要把组件的共享状态抽取出来，以一个全局单例模式管理。在这种模式下，我们的组件树构成了一个巨大的“视图”，不管在树的哪个位置，任何组件都能获取状态或者触发行为！另外，通过定义和隔离状态管理中的各种概念并强制遵守一定的规则，我们的代码将会变得更结构化且易维护。

不用 Vuex 会带来什么问题？
答：
一、可维护性会下降，你要想修改数据，你得维护三个地方
二、可读性会下降，因为一个组件里的数据，你根本就看不出来是从哪来的
三、增加耦合，大量的上传派发，会让耦合性大大的增加，本来 Vue 用 Component 就是为了减少耦合，现在这么用，和组件化的初衷相背。兄弟组件有大量通信的，建议一定要用，不管大项目和小项目，因为这样会省很多事。

Vue.js 中 ajax 请求代码应该写在组件的 methods 中还是 vuex 的 actions 中？
答： (关键看请求来的数据会不会复用)
一、如果请求来的数据是不是要被其他组件公用，仅仅在请求的组件内使用，就不需要放入 vuex 的 state 里。
二、如果被其他地方复用，这个很大几率上是需要的，如果需要，请将请求放入 action 里，方便复用，并包装成 promise 返回，在调用处用 async await 处理返回的数据。如果不要复用这个请求，那么直接写在 vue 文件里很方便。

怎么在组件中批量使用 Vuex 的 state 状态？
答：使用 mapState 辅助函数, 利用对象展开运算符将 state 混入 computed 对象中。

Vuex 中 action 和 mutation 有什么区别？
答：

  • action 提交的是 mutation，而不是直接变更状态。mutation可以直接变更状态。
  • action 可以包含任意异步操作。mutation只能是同步操作。
  • 提交方式不同，action 是用 this.$store.dispatch(‘actions方法名’,值) 来提交。mutation是用 this.$store.commit(‘commit方法名’,值) 来提交。
  • 接收参数不同，mutation第一个参数是state，而action第一个参数是context，其包含了

Vuex store 中的 state 是响应式的，响应规则：

  1. 提前在 store 中初始化好所需的属性
  2. 当给 state 中的对象添加新属性时：不能用[], 因为不是响应式的，用 Vue.set(obj, ‘newProp’, ‘newValue’);
  3. 当给 state 中的对象删除属性时：不能用 js 中的 delete obj.prop; 因为不是响应式的，要用 Vue.delete(obj, ‘prop’);

Vuex 的严格模式是什么,有什么作用,怎么开启？

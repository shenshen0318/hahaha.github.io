  3. 伪类和伪元素的区别
    答：css 引入伪类和伪元素的概念是为了格式化文档树以外的信息。也就是说，伪类和伪元素是用来修饰不在文档树中的部分。

    • 伪类用于当已有的元素处于某个状态时，为其添加对应的样式，这个状态是根据用户行为而动态变化的。
    • 伪元素用于创建一些不在文档树中的元素，并为其添加样式。它们允许我们为元素的某些部分设置样式。
    • 伪类的操作对象是文档树中已有的元素，而伪元素则创建了一个文档树外的元素。因此，伪类与伪元素的区别在于：有没有创建一个文档树之外的元素。
    • 具体看博客总结 CSS中的选择器及其权值

  4. css引入方式（4种）
    答：一共有4中引入方式

    • 内联方式：直接在元素标签（如div）内中使用css样式（结构与样式混合在一起，不易于代码维护，不建议使用。）
    • 嵌入方式：在 style 标签中书写css样式
    • 导入方式：使用 link 引入外部css文件
    • 链接方式：使用 @import 引入外部css文件
    • 追问：link 和 @import 的区别
      • link属于HTML标签，而@import完全是CSS提供的一种方式。link标签除了可以加载CSS外，还可以做很多其它的事情，比如定义RSS，定义rel连接属性等，@import就只能加载CSS了。
      • 加载顺序的差别。当一个页面被加载的时候，link引用的CSS会同时被加载，而@import引用的CSS会等到页面全部被下载完再被加载。所以有时候浏览@import加载CSS的页面时开始会没有样式，网速慢的时候页面会闪烁。
      • 兼容性的差别。由于@import是CSS2.1提出的所以老的浏览器不支持，@import只有在IE5以上的才能识别，而link标签无此问题。
      • 当使用javascript控制dom去改变样式的时候，只能使用link标签，因为@import不是dom可以控制的。
    • 行内样式优先级大于外部样式和内部样式，外部样式和内部样式谁先加载谁优先级高

  5. px、em、rem的区别
    答：

    • PX即像素，网页设计常用的基本单位。像素是相对显示器屏幕分辨率而言的。
    • em 是相对父元素的字体大小(参考物是父元素的font-size)。 如果当前父元素的字体尺寸未设置，则相对于浏览器的默认字体尺寸。如果自身定义了 font-size 按自身来计算。
    • rem 是css3 新单位，是相对于根元素html（网页）的字体大小（font-size）来计算的长度单位， 不会像em那样，依赖于父元素的字体大小，而造成混乱。
    • 具体看博客总结 CSS长度单位
    • 追问：使用 rem 布局的优缺点：
      • 优点：
        • 在屏幕分辨率千差万别的时代，只要将rem与屏幕分辨率关联起来就可以实现页面的整体缩放，使得在设备上的展现都统一起来了。
        • 而且现在浏览器基本都已经支持rem了，兼容性也非常的好。
      • 缺点：
        • 在奇葩的dpr设备上表现效果不太好，比如一些华为的高端机型用rem布局会出现错乱。
        • 使用iframe引用也会出现问题。
        • rem在多屏幕尺寸适配上与当前两大平台的设计哲学不一致。即大屏的出现到底是为了看得又大又清楚，还是为了看的更多的问题。
      • 追问：vw、vh 和 vm
        • css3新单位，viewpoint width/height 的缩写，视窗宽度/高度，1vw/vh 等于视窗宽度/高度的1%。

  7. margin塌陷及合并问题
    答：

    • margin塌陷：大盒子里面嵌套小盒子,如果这个父盒子里面没有文字，也没有边框（border），也没有padding-top （这三个条件中没有一个满足的）,给小盒子添加 margin-top 值,不但没有实现和大盒子之间的间距,反而传递到大盒子上,导致整体下移，这个现象就叫做margin值穿透。
      • 解决方案：
        • 给父级设置边框或内边距(不建议使用)
        • 给子元素的 margin-top 值改为给父元素加 padding-top 这样就避免了使用 margin-top 值导致的传递问题(转移了我们的目标对象)
        • 触发bfc(块级格式上下文),改变父级的渲染规则(最重要)
    • margin合并：如果两个同级的块级元素(两个盒子是上下关系),上边的一个盒子设 margin-bottom,下面的一个盒子设置 margin-top 两个盒子之间的距离并不是他们的margin之和,而是取最大的那个值。
      • 相邻元素的 margin-top 和 margin-bottom 会发生重叠（必须记住）空白内容的也会重叠；
      • 解决方案：
        • 只设置上面元素的margin-bottom
        • bfc（不常用，因为会改变html结构）
    • 具体看博客总结 margin相关问题
    • 追问：给行内元素设置padding和margin
    • 追问：百分比padding，margin是根据什么值计算的
      • 在默认的水平文档流方向下，CSS margin和padding属性的百分比值都是相对于父元素宽度计算的（不论是垂直方向上的还是水平方向上的）。

  8. display及其相关属性
    答：MDN把关键字取值分为了六类

    重点的有下面几个：

    • none：元素不显示，并从文档流中移除。
    • block：块类型。默认宽度为父元素宽度，可设置宽高，独占一行。
    • inline（默认）：行内元素类型。默认宽度为内容宽度，不可设置宽高，同行显示。
    • inline-block：行内块元素。默认宽度为内容宽度，可以设置宽高，同行显示。
    • table相关属性值：表格
    • inherit：继承。从父元素继承display属性的值。
    • list-item：像块类型元素一样显示，并添加样式列表标记。
    • flex：弹性布局
    • grid：网格布局
